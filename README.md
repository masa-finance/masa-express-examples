## Examples

### [Soul Names](./examples/soul-names)

This example implementation will run a service based on Masa Express that fulfills the following requirements:

- the service will be compatible with the Masa SDK. It will be able to log in to the endpoint generated by this service.
- the service will store the Session information in Memory
- the service will generate a basic image and Soulname Metadata that is compatible with OpenSea.
- the service will persist the metadata in Arweave
